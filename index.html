Natural Language Processing

(NLP)

Some ideas here come from the talk by Sebastian Dziadzio @sebadzia Watch his pyData talk on NLP

What is it?

A set of techniques and tools to turn the language we speak, write or sign into machine interpretable representations.

In other words: Text -> Numbers

Not quite. This will let the computer read the letters, but will not convey any information about the meaning, grammar or structure of the text.

We are trying to preserve much more information and convey it to the computer in a more meaningful way for analysis.

It is hard to get the `right` numbers

  1. Language is a product of evolution.
    • e-liquid
    • mansplain
    • ransomware
    • deglobalization
    • hangry
  2. Massive issues surrounding syntactic, phonetic and semantic ambiguity
    • "Time flies like an arrow."
    • "Fruit flies like a banana."
  3. And all the lovely issues that we had in the `Data` section (abbreviations etc.)
The list there is a sample of the the Oxford dictionary new word list for 2018.

With regards to point 2 we can really see from the data I've shown before that its not always clear what is what. To give you a more 'well behaved' example of ambiguity here are two sentences with the same words meaning different things.

The tools and techniques used for the purposes of solving these issues are are numerous.

It is a very vast field.
